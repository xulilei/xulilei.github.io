入职强化学习之MyBatis

什么是 MyBatis？

MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作。MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

Mybatis与Hibernate的区别？

Hibernate是一种全自动ORM框架，它旨在消除编写SQL的过程，将JDBC的全过程封装起来，但是对定制SQL，优化SQL的支持不足。

Hibernate是全映射的框架，数据库有多少条字段，那么javaBean就有多少字段。

Mybatis做的是？

SQL语句交给开发人员编写，其余过程交给框架。

Mybatis的核心文件编写

1、POJO映射数据库表的Java对象

2、Dao：用于操作数据库的接口

3、sql mapper：sql映射配置文件，用于注册接口，使其具有操作数据库的能力

4、mybatis配置文件，初始化一些基本的配置信息，用于生成sqlSessionFactory

Mybatis核心代码编写步骤

1、创建项目添加Maven依赖

2、创建数据库表

3、映射数据库表的POJO对象

4、操作数据库的映射器接口类

5、映射数据库的SQL Mapper XML文件

6、mybatis的核心配置文件，包含数据库的基本信息以及mybatis的核心配置等

SqlMapper配置文件-mapper.xml

具体参数

参数名称 参数意义
namespace 被注册的接口
id 方法名
parameterType 入参类型（类要写全限定类名）
resultType 返回值类型
#{} 可以直接获取实体类参数
useGeneratedKeys 提取由数据库内部生成的主键（仅对insert和update有用）
keyProperty 对应实体类的主键名称

Mybatis核心配置文件-mybatis-config.xml

注意点

1、初始化数据库的基本信息

2、注册映射数据库的SQL map配置文件

3、mybatis的其他一些配置

4、与spring整合时移到spring的配置文件中

重要参数

参数名称 作用
properties 引入外部配置文件，使用${}获取外部参数
settings 调整设置，详见下表
typeAliases 别名代替全类名，还可以指定包名下的所有类
mappers 配置sqlMapper映射文件的

settings设置属性

名称 作用
mapUnderscoreToCamelCase 开启后，数据库命名规则自动映射java驼峰命名对象，如user_name->userName
